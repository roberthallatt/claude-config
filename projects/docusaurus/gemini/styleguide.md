# Docusaurus Code Style Guide

## Key Principles

- **Clarity**: Clear, concise documentation
- **Consistency**: Uniform structure and formatting
- **Accessibility**: WCAG compliant content
- **Examples**: Practical code examples

## Markdown

- Use descriptive front matter
- Hierarchical headings (h2 → h3 → h4)
- Use admonitions for callouts
- Include working code examples

## React Components

- TypeScript for type safety
- Document component props
- Keep components focused

## Styling

- Support both light and dark mode
- Use CSS variables for theming
- Consistent spacing

## Code Review Focus

1. Documentation clarity
2. Correct MDX syntax
3. Accessibility
4. Code example accuracy
5. Proper categorization
