# Project Instructions

## Tech Stack

This is a **Docusaurus** project using:
- **Docusaurus** - Documentation framework
- **Markdown/MDX** - Content format
- **React** - Components
- **TypeScript** - Type safety

## Project Structure

```
project-root/
├── docs/           # Documentation
├── blog/           # Blog posts
├── src/
│   ├── components/
│   └── pages/
├── static/         # Static assets
└── docusaurus.config.js
```

## Development

```bash
npm run start     # Dev server
npm run build     # Build
npm run serve     # Serve build
```

## Conventions

1. Use descriptive front matter
2. Hierarchical headings
3. Code examples with syntax highlighting
4. Admonitions for callouts
5. MDX for interactive content
